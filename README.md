## Таблица пользователей (тестовое задание JS/React)

Приложение реализовано на **React 18+** и **Vite**. Оно отображает таблицу пользователей с сервера DummyJSON с сортировкой, фильтрацией, пагинацией, модальным окном с подробной информацией и изменяемой шириной колонок.

### Функциональность

- **Таблица пользователей**: колонки фамилия, имя, отчество, возраст, пол, номер телефона, email, страна, город.
- **Сортировка с помощью HTTP-запросов**:
  - Используется `https://dummyjson.com/users?sortBy=...&order=...`.
  - Три состояния сортировки на клике по заголовку: по возрастанию, по убыванию, без сортировки.
  - Поддерживаемые поля сортировки: фамилия (`lastName`), имя (`firstName`), отчество (`maidenName`), возраст (`age`), пол (`gender`), телефон (`phone`).
- **Фильтрация**:
  - По ФИО (одно текстовое поле).
  - По полу (select).
  - По стране, городу, email и телефону (отдельные текстовые поля).
  - Фильтрация выполняется на клиенте по данным, полученным с сервера.
- **Постраничное отображение**:
  - Используется `limit` и `skip` параметрами DummyJSON.
  - Размер страницы: 10 пользователей.
  - Отображается текущая страница и общее количество страниц/пользователей.
- **Модальное окно с подробной информацией**:
  - Открывается по клику на строку таблицы.
  - Содержит ФИО, возраст, пол, рост, вес, телефон, email, полный адрес и аватар пользователя.
- **Обработка ошибок**:
  - Сообщение об ошибке при неуспешном HTTP-запросе.
  - Кнопка "Повторить попытку" для перезапроса данных.
- **Изменяемая ширина колонок**:
  - У каждого заголовка справа есть «хендл» для растягивания.
  - Минимальная ширина столбца — 50px.
- **Адаптивная верстка**:
  - Ширина таблицы — 100% viewport'а, но не более **1400px**.
  - Верстка оптимизирована под десктоп и небольшие экраны.

### Технологический стек

- **React** `^18` (фактически React 19, совместимый с требованием React 18+).
- **Vite** как сборщик.
- Чистый **Fetch API** для работы с HTTP-запросами, без сторонних HTTP-библиотек.

### Структура проекта (основные файлы)

- `src/App.jsx` — корневой компонент приложения, хранит состояние фильтров, сортировки, пагинации и выбранного пользователя, собирает все части UI.
- `src/hooks/useUsers.js` — кастомный хук для загрузки пользователей с DummyJSON с учетом страницы и сортировки.
- `src/components/UserTable.jsx` — таблица пользователей: сортировка, вывод данных, обработка кликов по строкам, ресайз колонок.
- `src/components/UserModal.jsx` — модальное окно с подробной информацией о пользователе.
- `src/App.css` — стили для компонента приложения, таблицы, пагинации, модального окна.
- `src/index.css` — глобальные стили (фон, шрифты, базовая типографика).

### Как запустить локально

1. Установите зависимости:

   ```bash
   npm install
   ```

2. Запустите дев-сервер:

   ```bash
   npm run dev
   ```

3. Откройте указанный в консоли адрес (обычно `http://localhost:5173`).

### Сборка приложения

```bash
npm run build
```

Собранные файлы будут лежать в директории `dist`.

### Размещение на GitHub и GitHub Pages

1. **Создайте репозиторий** на GitHub и свяжите локальный проект:

   ```bash
   git init
   git add .
   git commit -m "Initial commit: users table app"
   git branch -M main
   git remote add origin git@github.com:<your-username>/<your-repo>.git
   git push -u origin main
   ```

2. **Настройте GitHub Pages** для Vite:

   - В `vite.config.js` при необходимости установите `base` (например, `/your-repo/`), если приложение будет лежать не в корне домена:

   ```js
   // vite.config.js
   export default defineConfig({
     plugins: [react()],
     base: '/your-repo/', // заменить на имя репозитория
   })
   ```

3. **Соберите проект и выложите в ветку `gh-pages`** (один из вариантов — использовать `gh-pages` пакет, другой — Action'ы GitHub):

   Пример через `gh-pages`:

   ```bash
   npm install --save-dev gh-pages
   ```

   В `package.json` добавьте скрипты:

   ```json
   {
     "scripts": {
       "predeploy": "npm run build",
       "deploy": "gh-pages -d dist"
     }
   }
   ```

   Затем:

   ```bash
   npm run deploy
   ```

   Это создаст/обновит ветку `gh-pages` с собранным приложением.

4. **Включите GitHub Pages** в настройках репозитория:

   - Зайдите в `Settings` → `Pages`.
   - В разделе `Branch` выберите ветку `gh-pages` и папку `/ (root)`.
   - Сохраните изменения — через некоторое время приложение станет доступно по URL вида:
     `https://<your-username>.github.io/<your-repo>/`.

### Проверка задания

- Убедитесь, что:
  - Приложение собирается без ошибок (`npm run build`).
  - Сортировка реально меняет параметры запроса (см. Network в DevTools).
  - Фильтры, пагинация, модальное окно и ресайз колонок работают корректно.
  - Приложение доступно по ссылке GitHub Pages, и ссылка добавлена в описание репозитория.
